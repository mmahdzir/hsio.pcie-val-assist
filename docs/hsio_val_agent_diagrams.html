📝 Sequence File

  • Full port iteration loop (PCH_MIN_VAL_MODEL guards)
  • Call base tasks: dpc_configure_base_regs(), etc.
  • Phase-by-phase implementation from scenario .txt
  • Apply model-specific fixes (SoC fork timeout, x8 return)
  • Use correct macros: pch_PCIE_REG_SET / pch_PCIE_REG_WRITE
  • Use sideband for RO registers: pch_PCIE_REG_MEM_WRITE

🏷️ Test Class File
subsystems/pcie/verif/tests_vegan/<testname>/<testname>.svh
Minimal boilerplate — the test class just wires up the sequence:
class <testname> extends pch_pcie_base_test;
  `uvm_component_utils(<testname>)
  function new(string name, uvm_component parent);
    super.new(name, parent);
    user_data_seq_name = "<testname>_seq";
  endfunction
endclass
  • Extend pch_pcie_base_test
  • Set user_data_seq_name to wire the sequence
  • Register with `uvm_component_utils
  • Directory name = class name = file name (no mismatch)

📂 Stage 2 — Log Collection: Extract Evidence
🗒️ Simulation Logs
  • postmortem.log — OVERALL PASS/FAIL, UVM_ERROR/FATAL count
  • jestr.log — full sim log; search for PCIE_TEST_INFO, UVM_ERROR, UVM_FATAL, phase transitions
Key grep patterns:
grep "UVM_FATAL\|UVM_ERROR" jestr.log grep "PCIE_TEST_INFO.*Phase" jestr.log grep -o "@ [0-9]*" jestr.log | sort -t@ -k2 -n | tail -3
🔬 TLP & Fabric Trackers
  • PCIe0_0_trk.out — BFM TLP tracker: TLP types, EP bit, LTSSM states
  • IOSFP_NORTH_INTF_*.trk — IOSF Primary fabric: CfgRd/CfgWr, MWr, CplD, MSI
  • IOSFSB_FAB_TRK_node_PCIE.log — IOSF Sideband: register config via SB
Key checks:
  • EP bit = 1 on poisoned TLP? Check last field before CRC
  • MSI MWr32 to 0xFEExxxxx present?
  • CplD with UR/CA status returned?
  • LTSSM → Disabled after DPC?
📡 Waveform (FSDB)
dump.fsdb
Use Verdi or nWave to inspect signal-level behavior when log analysis is insufficient:
  • DPCSR.DPCTS assertion timing
  • LTSSM state machine transitions
  • CLKREQ / CLKACK gating (SB clock idle = normal)
  • PCIe link status signals
⚠️ Decompress .fsdb.gz before opening — compressed is 5+ min to load

evidence gathered
🏷️ Stage 3 — Failure Classification
🛑 Hang / Timeout
  • Sim did not complete within time limit
  • Jestr shows no progress at later sim times
  • Check last PCIE_TEST_INFO message → which phase?
  • SoC: BFM block on disabled link → need fork timeout
  • SoC TOM: POWER_GOOD_PHASE timeout → boot model config issue
  • X4/X8: SLA_MAX_RUN_CLOCK too small?
❌ UVM_ERROR / UVM_FATAL / Wrong Result
  • UVM_FATAL → always real, fix before anything else
  • UVM_ERROR from SBR_INFO PATH_CHECK → SoC noise, ignore
  • DPC register mismatch → check register config chain order
  • Wrong TLP / missing MSI → check trackers for mis-routing
  • DPCTS not set → AER chain incomplete, check all 13 registers
  • DSTS.CED = 0 on PXPB → expected after single-port DPC recovery
Failure Signal Severity First Check Common Root Cause
UVM_FATALCRITICALgrep UVM_FATAL jestr.logAssertion, null handle, phase timeout
OVERALL FAIL + UVM_ERRORHIGHgrep UVM_ERROR jestr.logScoreboard mismatch, reg value wrong
Sim hang (no UVM_FATAL)HIGHLast sim time + phaseBFM block, infinite wait, missing event

SBR_INFO PATH_CHECKNOISECount > 300?Normal SoC sideband routing — waive

🔧 Stage 5 — Root Cause Analysis & Fix
Common Root Cause Categories:
Sequence logic error
Wrong register value, missing reg in chain, wrong phase order. Fix: edit *_seq.svh, recompile+elab.
Model-specific incompatibility
SoC: PIBAItems → GetTxs(), BFM hang → fork timeout. Fix: apply SoC guards, recompile+elab.
Register access type mismatch
Writing RO register via primary access. Fix: switch to sideband macro or STRPFUSECFG.
Multi-port issue (x8)
Test runs on port 1 causing unexpected behavior. Fix: add return; after first port.
Infrastructure / environment
Wrong WORKAREA, stale touch files, cleanup_junk exit 127. Fix: env check, delete touch, verify postmortem separately.
Expected behavior misclassified
SBR_INFO PATH_CHECK (SoC noise), DSTS.CED=0 (PXPB recovery). Fix: document as waived.
When to Recompile vs Just Rerun:
Change MadeRequired Action
Sequence .svh modifiedCompile + Elab + Rerun
Test class .svh modifiedCompile + Elab + Rerun
base_seq.svh modifiedCompile + Elab + Rerun ALL
*.inc include file modifiedCompile + Elab + Rerun

Random seed changedJust rerun
RTL file modifiedFull rebuild from defacto

Per-Test Result Collection
for testdir in "$REGBASE"/<testlist_dir>/pch_*; do
  grep "OVERALL" "$testdir/postmortem.log" 2>/dev/null | head -1

Collect for each test:

Test Name Matching Rules
  • SoC tests have controller suffix: _PXPA, _PXPB, _PXPC, _PXPD, _SOC_PXP*
  • g5s3 tests may have _IOE suffix or plain names

⚠️ Critical matching rule:
Do NOT match pch_pcie_L1_Off to pch_pcie_L1_Off_genMax — these are DIFFERENT tests. Exclude matches where the regression name is a prefix of a longer testplan entry.
